# Использование kubectl изнутри кластера в поде

## Описание.
Необходимо для работы с компонентами k8s из пода. Пригодится для автоматизации задач.

## Поехали!
```
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: ServiceAccount
metadata:
  annotations:
    kubernetes.io/enforce-mountable-secrets: "true"
  name: kubectl-access
  namespace: default
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: kubectl-access
  namespace: default
rules:
- apiGroups: ["*"]
  resources: ["*"]
  verbs:
  - create
  - get
  - list
  - watch
  - update
  - patch
  - delete
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: kubectl-access
  namespace: default
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: kubectl-access
subjects:
- kind: ServiceAccount
  name: kubectl-access
  namespace: default
EOF
```

### Под.
```
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: kubectl
  namespace: default
  labels:
    test: kubectl
  annotations:
    author: cameda
spec:
  containers:
  - name: kubectl
    image: bitnami/kubectl
    imagePullPolicy: IfNotPresent
    resources:
      requests:
        cpu: "50m"
        memory: "50Mi"
      limits:
        memory: "100Mi"
    command: ["sh", "-c"]
    args: ["sleep 3650d"]
  restartPolicy: OnFailure
  hostname: kubectl
  serviceAccountName: kubectl-access
  nodeSelector:
    kubernetes.io/os: linux
EOF
```
